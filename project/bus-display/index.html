Background

Problem & Purpose

問題背景與目的

Hong Kong's public bus network spans hundreds of routes operated by KMB and CTB. The official apps are functional, but heavyweight — requiring account creation, location permissions, and multiple taps to reach a live arrival time for a familiar stop.

香港公共巴士路線眾多，官方應用程式功能完善，但操作繁複，往往需要登入帳戶及多次點擊才能查看到站時間。

This project is a personal-use tool that loads instantly in a mobile browser, remembers your favourite stops across sessions with no backend, and surfaces the next three departure times within two taps. It uses only open government APIs — no scraping, no unofficial data sources, no user tracking.

本工具在手機瀏覽器即開即用，無需後端，透過 localStorage 記住常用站點， 兩次點擊即可顯示下三班車到站時間，並僅使用香港政府開放交通數據。

Audience

Target User

目標用戶

A daily commuter in Hong Kong who takes the same 2–4 bus routes every day. They know their routes and stops already — they just want the next ETA without friction, on a phone browser, without installing anything or creating an account.

每天乘搭固定路線的香港通勤者。他們熟悉路線和站點，只需快速查看下一班車時間， 無需安裝應用程式或登入帳戶。

The saved board persists in localStorage, so their common stops reload immediately on every visit — without any server, account, or sync mechanism.

Design

Key UX Decisions

主要設計決策

  • 1 Fixed-height stop tiles. Each stop row has a fixed height and never reflows neighbours when it expands — only the expanded tile itself grows, preventing disorienting layout jumps during a live ETA fetch.
  • 2 Numpad-first input. A phone keyboard covers half the screen and interrupts the route list. A custom numpad keeps the full list visible while typing, and the prefix trie dims invalid keys in real time — no mis-taps, no dead ends.
  • 3 Parallel ETA fetch. All stop ETAs for a route are fetched simultaneously via Promise.allSettled, not sequentially. The stop list renders immediately with loading states, then updates in place once all fetches resolve — typically under two seconds.
  • 4 Stale token cancellation. Each direction-switch increments an etaFetchToken. When a fetch resolves, it checks if the token still matches — stale responses from previous direction selections are silently discarded.
  • 5 Opposite-direction tile. A single tap at the bottom of the stop list flips to the return direction without going back to search. This cuts three taps to one for commuters who switch direction frequently.

  • Zero dependencies — no frameworks, no build tools. One HTML file, self-contained CSS and JS.
  • Prefix trie — built in-memory from 797 KMB routes on load, extended live as CTB routes arrive.
  • Route registry — a Map<company:route, variants[]> built from both APIs, used for destination labels and direction logic.
  • KMB bulk stop cache — 4,000+ stops fetched once and stored in a Map for instant name lookups. CTB stops are fetched individually and cached per session.
  • 15-second refresh loop — setInterval on the board screen; cleared whenever navigating away.
  • localStorage persistence — board JSON serialised on every change; loaded synchronously on startup.
  • Offline banner — listens to online/offline events and shows a fixed banner when connectivity is lost.

Engineering

Technical Decisions

技術選擇

Decision Rationale
Vanilla JS, no framework Single-file constraint eliminates any build step. DOM is simple enough that framework overhead would outweigh benefit.
Hardcoded KMB route list (797 routes) KMB route numbers are stable. Embedding them allows the prefix trie to function instantly on load, before any API call completes.
Promise.allSettled for all parallel fetches Partial failures (e.g. one stop returning 404) should not abort the entire ETA display. Each stop resolves independently.
etaFetchToken (integer counter) Async direction-switch creates a race condition. A monotonic token means only the most recent fetch's result is applied.
KMB bulk /stop endpoint cached once Fetching 4,000+ stop names individually would take too long and exceed rate limits. One bulk fetch on first use, then instant Map lookups.
CTB individual stop fetches, cached per session CTB has no bulk-stop endpoint in the public API. Per-stop fetches are parallelised and memoised in ctbStopCache to avoid repeated requests.
localStorage, no backend Eliminates auth, server costs, and privacy concerns entirely. Acceptable trade-off for a personal-use tool where one browser is the primary client.
CSS-only responsive layout, no media queries for core flow The three-screen model uses 100dvh per screen, so it works on any viewport without JS viewport detection. Media queries only adjust density for landscape.

Trust

Privacy & Security

私隱及安全

This app collects nothing about you.

本工具不收集任何用戶資料。

  • No account creation, no login, no email required.
  • All preferences stored in your own browser's localStorage only — never sent anywhere.
  • No analytics scripts, no tracking pixels, no third-party scripts of any kind.
  • All data comes from open Hong Kong Government APIs (KMB ETA API, Citybus NWFB API) — the same data used by official apps.
  • Clearing your browser's site data removes all saved state entirely.

Honest Assessment

Constraints

限制說明

  • Requires internet. All ETA data is fetched live from public APIs. The app does not function offline beyond showing cached board layout.
  • ETA accuracy depends on the API. Arrival predictions are provided by the bus operators via government APIs. Delays, cancellations, or missing data reflect the upstream source.
  • localStorage cleared = data lost. There is no cloud sync or backup. Clearing browser storage or using a new device requires rebuilding your board from scratch.
  • CTB individual stop fetches on first load. CTB has no bulk-stop endpoint, so loading a CTB route's stops requires one API call per stop. This is noticeably slower than KMB on first visit, though cached for the session.
  • Single-user, single-browser design. Designed for personal use on one device. Not intended as a multi-user or shared application.
